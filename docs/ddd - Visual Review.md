# <span style="color:#6D5BD0"><u><strong>SpritesPlayground DDD</strong></u></span>

Source of truth: [[New PRD]]. Genesis authoring-scene design lives in [[New PRD Genesis]]. This document converts the PRD into a domain, data, runtime, and validation design. It intentionally does not define UI layout. It marks user-authored fields so Genesis can edit stable domain responsibilities instead of ad hoc presentation assumptions.

## <span style="color:#2F80A8"><u><strong>1. Product Domain Overview</strong></u></span>

### <span style="color:#4F9D69"><u><strong>1.1 Domain Purpose</strong></u></span>

SpritesPlayground is a constrained 2D pixel ACT playground for classic belt-scroll arcade action. Its domain is not engine creation; its domain is repeatable production of playable, swappable pixel-action sprites with clear runtime ownership.

| Area | DDD Classification | Reason |
| --- | --- | --- |
| Sprite action runtime | Core domain | Defines the playable feel: state, input, MoveData timing, combat, hit reaction, and visual sync. |
| Sprite definition data | Core domain | CharacterTemplate, MoveData, MoveSegment, SpriteSet, and generated SpriteFrames are the stable production contract. |
| Combat resolution | Core domain | Damage, hit-once, <strong><span style="color:#C65D5D">hitstop</span></strong>, hitstun, and hurt reaction are game-specific rules. |
| Direction and visual coverage | Core domain | Direction policy controls production cost and runtime clip resolution. |
| Asset generation | Supporting domain | Materializes mirrored clips, mirrored hitboxes, and runtime SpriteFrames from source data. |
| Validation pipeline | Supporting domain | Protects contracts before runtime and before future tooling writes data. |
| Godot primitives | Generic domain | CharacterBody2D, Area2D, CollisionShape2D, AnimatedSprite2D, SpriteFrames, AnimationPlayer, JSON/FileAccess. |
| LimboAI orchestration | Generic/supporting domain | Provides HSM/BT/Blackboard execution; project data remains separate. |

### <span style="color:#4F9D69"><u><strong>1.2 Explicit Non-Domain</strong></u></span>

| Non-Domain | Rule |
| --- | --- |
| General 2D engine | Forbidden by PRD. No reusable state graph engine, combat framework, or asset editor. |
| Raw art editing | Crop, slicing, palette editing, and paint edits happen in external art tools. |
| Runtime production mirroring | Mirrored clips and hitboxes are generated before runtime. |
| Full combo-tree engine | Out of baseline. The baseline only supports linear segmented continuation such as <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">AAA</code> / <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">JJJ</code>, plus first-token-disjoint moves such as <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">AAA</code>, <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">BBB</code>, <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">CBBB</code>. Same-prefix branching such as <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">AAA</code> vs <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">AAB</code> is <strong><u><span style="color:#C65D5D">forbidden</span></u></strong> for now. |
| Multiplayer | Out of baseline scope. |
| Weapon/projectile framework | Out of baseline scope. |
| Genesis UI layout | Owned by [[New PRD Genesis]]. This DDD marks data responsibility only and <strong><u><span style="color:#C65D5D">must not</span></u></strong> duplicate screen layout. |

### <span style="color:#4F9D69"><u><strong>1.3 Domain Boundary Statement</strong></u></span>

```md
CharacterTemplate + MoveData + SpriteSet
-> generated runtime assets
-> runtime actor instance
-> StateDriver / Behavior / MoveRuntime
-> CombatResolver
-> VisualPresenter
-> ValidationReport
```

Authoring data is persistent truth. RuntimeBlackboard is instance truth. Generated SpriteFrames are rebuildable runtime assets. Animation names never define gameplay. Only [[New PRD]] and this DDD are current design truth; historical PRDs, v0.3 data, and live legacy runtime values are reference only.

| <code style="color:#2F80A8;font-weight:700;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">MoveData</code> | Persistent committed-action data: category, state event, direction policy, input command, ordered segments, hitboxes, segment gates, finish event. | A move may be single-segment or segmented; it is not a node tree or generic behavior label. |

| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">SegmentGate</code> | Frame window that accepts buffered input to continue a segmented MoveData. | Linear continuation only in baseline; no same-prefix branching tree. |
| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">Behavior</code> | Executable state behavior or MoveData-backed action selected by StateDriver/BTState. | Behavior coordinates execution; source-of-truth data remains in CharacterTemplate/MoveData/SpriteSet. |
| <code style="color:#7C65C1;font-weight:700;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">State</code> | Control/physics context such as idle, walk, jump, fall, attack, hurt, dead. | Does not encode direction, skin, combo branch, attack type, or damage. |
| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">StateDriver</code> | LimboAI HSM/BT owner for legal state transitions and state behavior execution. | Does not own raw input, damage formula, or animation assets. |
| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">RuntimeBlackboard</code> | Runtime instance key/value state: current state, current move, facing, velocity, HP, input buffer, hurt result. | Runtime-only; not persisted as sprite-definition truth. |
| <code style="color:#2F80A8;font-weight:700;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">InputSystem</code> | Actor-local adapter that reads Godot input and produces intent, input buffer, command sequence, and combo string. | Does not own transitions or damage. |
| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">MoveRuntime</code> | Executes the current MoveData frame timeline: startup, active, recovery, cancel, <strong><span style="color:#C65D5D">hitstop</span></strong>, frame events. | Does not traverse targets or mutate HP. |
| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">CombatPorts</code> | Actor-local hitbox source and hurtbox receiver ports. | Exposes samples; does not resolve final damage. |
| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">CombatResolver</code> | Stage service that resolves attacker-target traversal, intersections, hit-once registry, damage formula, and HurtResult dispatch. | Damage is a world relation, not actor-local. |
| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">LifeRuntime</code> | Actor-local HP owner applying final damage and emitting death. | Does not own ATK, DEF, invincibility, armor, <strong><span style="color:#C65D5D">hitstop</span></strong>, hurt animation, or bars. |
| <code style="color:#4F9D69;font-weight:700;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">SpriteSet</code> | Persistent visual coverage data: action clips, direction coverage, frame sequences, generated mirror provenance. | Does not own damage math or HP. |
| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">VisualPresenter</code> | Runtime visual adapter resolving clip names, direction policy, frame sync, <strong><span style="color:#C65D5D">hitstop</span></strong> freeze, feedback, debug overlay. | Presentation only; never gameplay truth. |
| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">GeneratedSpriteFrames</code> | Rebuildable Godot SpriteFrames/clip resources produced from SpriteSet and preprocessing. | Not source of truth. |
| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">HitboxProfile</code> | Move-owned active attack sample profile with ATK, rect, active window, <strong><span style="color:#C65D5D">hitstop</span></strong>, hitstun, knockback, reaction tag. | Belongs to MoveData. |

| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">DirectionPolicy</code> | Rule for authored/generated facing coverage: <code style="color:#4F9D69;font-weight:700;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">single_south</code>, <code style="color:#2F80A8;font-weight:700;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">side_2flip</code>, <code style="color:#D48A33;font-weight:700;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">full_8flip</code>, <code style="color:#7C65C1;font-weight:700;font-style:italic;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">custom</code>. | Selected by action/move; direction never creates State branches. |
| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">FrameTimeline</code> | Authored frame sequence shared by animation beats, hitboxes, hurtboxes, SFX/VFX, <strong><span style="color:#D48A33">cancel windows</span></strong>, and recovery. | Uses authored frames; runtime tick checks can be 60 Hz. |
| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">ComboString</code> | Buffered symbolic input string such as <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">AAA</code> / <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">JJJ</code> used to satisfy MoveSegment gates. | Not hit-count display and not a branching combo-tree engine. |
| <code style="color:#475467;font-weight:500;background:#F3F6FA;border:1px solid #D9E2EC;border-radius:4px;padding:0 0.18em">Category</code> | MoveData classifier such as movement/combat/defense. | Does not decide whether something is a State. |

### <span style="color:#4F9D69"><u><strong>4.2 Ownership Rules</strong></u></span>

| Rule | Contract |
| --- | --- |
| Character body ownership | CharacterTemplate owns size class, base hurtboxes, DEF, foot/body collision, HP baseline, equipped move refs, SpriteSet ref. |
| Move ownership | MoveData owns timing, category, input command, segments, segment gates, hitbox ATK profiles, <strong><span style="color:#C65D5D">hitstop</span></strong>/hitstun metadata, finish event. |
| Visual ownership | SpriteSet owns clip coverage, direction coverage, frame sequences, slot status, generated mirror provenance. |
| Runtime ownership | RuntimeBlackboard owns only instance values. It may reference source IDs but cannot redefine source data. |
| Combat ownership | CombatResolver owns target traversal, hit-once registry, damage formula, HurtResult dispatch. |
| HP ownership | LifeRuntime owns current/max HP at runtime and applies final damage only. |
| Presentation ownership | VisualPresenter resolves and displays. It cannot create gameplay truth. |

### <span style="color:#4F9D69"><u><strong>7.2 Input to Move Execution</strong></u></span>

```md
Godot Input / AI intent
-> InputSystem reads edges at runtime tick rate
-> input_buffer records recent presses/releases
-> InputSystem maps physical key to input action, e.g. J -> attack_light
-> if not already attacking: input_intent emits ev_attack
-> StateDriver validates transition into attack context
-> BTState selects MoveData
-> MoveRuntime starts MoveData segment 0
-> if already attacking: buffered input may emit MoveCommand advance_segment
-> VisualPresenter resolves clip by state/move/segment + facing + direction policy
```

### <span style="color:#4F9D69"><u><strong>7.3 MoveRuntime Frame Execution</strong></u></span>

```md
MoveData selected
-> move_instance_id generated
-> segment 0 starts
-> segment_attack_instance_id generated
-> authored segment frame counter starts at 0
-> startup frames tick
-> active_window exposes HitboxSource samples
-> segment gate opens by authored frame
-> buffered matching input may advance to next segment
-> each new segment gets a new segment_attack_instance_id
-> final segment recovery completes
-> finish_event = ev_finished
-> StateDriver returns to idle / walk
```

### <span style="color:#4F9D69"><u><strong>7.4 Hit Resolution</strong></u></span>

```md
HitboxSource.active_samples
-> CombatResolver target traversal
-> HurtboxReceiver.effective_hurtboxes
-> intersect hitbox_rect with hurtbox_rect
-> reject invalid target / invincible / already hit
-> choose hurtbox by priority, smallest area, registration order
-> selected_hurtbox_def = hurtbox_def or 0
-> final_damage = max(0, hitbox_atk - selected_hurtbox_def)
-> LifeRuntime.apply_damage(final_damage)
-> HurtboxReceiver builds HurtResult
-> RuntimeBlackboard.hurt_result = HurtResult
-> LifeRuntime emits observable ev_dead if HP <= 0
-> StateDriver dispatches ev_hurt or consumes ev_dead
-> VisualPresenter plays feedback
```

### <span style="color:#4F9D69"><u><strong>7.5 Death Flow</strong></u></span>

```md
LifeRuntime.current_hp reaches 0
-> LifeRuntime emits observable ev_dead
-> StateDriver consumes ev_dead and transitions to dead
-> MoveRuntime current move is cancelled
-> CombatPorts stop exposing offensive samples
-> VisualPresenter resolves dead clip
-> RuntimeBlackboard.current_state = dead
```

### <span style="color:#4F9D69"><u><strong>7.6 SpriteSet Clip Resolution</strong></u></span>

```md
state_or_move + behavior_id + facing_dir
-> action key
-> DirectionPolicy selected
-> authored facing or generated facing resolved
-> GeneratedSpriteFrames clip name resolved
-> frame index synced to authored frame timeline
-> warning if missing / placeholder / unsupported
```

Runtime <strong><u><span style="color:#C65D5D">must not</span></u></strong> silently substitute clips. If a clip is missing, the resolver returns explicit warning status and a debug fallback only if configured by system rules.

### <span style="color:#4F9D69"><u><strong>7.7 Runtime Validation Flow</strong></u></span>

```md
-> move timing validation
-> segment gate validation
-> hitbox/hurtbox ownership validation

```

## <span style="color:#2F80A8"><u><strong>9. Invariants and Validation Rules</strong></u></span>

| GeneratedSpriteFrames are rebuildable. | Reject manual source-of-truth edits to generated assets. |
| Only [[New PRD]] and [[ddd]] are current design truth. | Historical PRDs, v0.3 data, and live legacy code cannot override these contracts. |

| Rule | Validation |

### <span style="color:#4F9D69"><u><strong>9.4 Combat Invariants</strong></u></span>

| Rule | Validation |
| --- | --- |
